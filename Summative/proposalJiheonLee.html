Study Helper

Jiheon Lee

G O A L

The Study Helper contains three main functions to assist students' study life in a efficient and visual way.
With the Digital Clock including a stopwatch and an alarm, it suggests an alternative timing tool instead of a phone.
With the Flashcards, it efficiently helps students memorize various types of questions.
With the ToDo List, it allows students organize their daily schedule and help planning tasks.

Target User

1. Middle school or high school students who need to strengthen their study habit.

2. Students preparing tests which has time limits

3. Students who need to memorize vocabularies

Total Time

Flowcharts

[Digital Clock]

1. Real-Time Clock

        This flowchart shows how the clock works.
Once the start button is clicked, the function will executed every 1 second.
        In the function, it first get the information of current day, hour, minute, and second. To format the hour which is in 24 hours, ut determines the period using the conditional and the remainder of hours divided by 12. Then, ut connects the strings of hours, minutes, and seconds to display on the clock. Lastly, it displays day and period on the clock.

2. Stopwatch

        This flowchart shows the basic operation of the stopwatch.
Once the start button is clicked, it initialize an array of 8 digits, which is on the page. Then, the function is called every 10 milliseconds.
        With a loop iterating the last 7 digits of the array, it checkes if any digit turns to 10. If so, it resets the digit to 0 and increases the next digit by 1. Similarly, the loop iterates from the tenth digit of seconds to the tenth digit of minutes to check if any digit turns to 6. If so, it resets the digit to 0 and increases the next digit by 1. After checking them, it increases the last digit, the oneth digit of millisecond, by 1 to make the stopwatch flow.

3. Alarm

        This flowchart shows the basic operation of the alarm.
First, the program gets a string of a series of digits from user input. With this input, the function divides it into three parts - hours, minutes, and seconds. Then, it multiplies 3600 to the hours and 60 to the minutes to calculate the total seconds by adding the values. After determining the total seconds left, it calls the function every 1 second.
        In the function, the total seconds decreases by 1, and the function convert this total seconds into a six-digit string. Then, it displays the updated string on the page. Meanwhile, when the total seconds was calculated, it automatically countdowns and rings the alarm as well as flashes the background at last.

[Flashcards]

1. Adding Cards

        This flowchart shows how cards are added to flashcards.
When the user enter a question ans an answer, the function push the values in two arrays for questions and answers.
        Then, it increases the integer variable of total number of cards by 1 and changes the number of current card to show to the last number. For the next input, it resets the input fields and closes the element. Finally, it display the newly added card with the question and the answer stored in the arrays.

2. Deleting Cards

        This flowchart shows how cards are deleted from flashcards.
First, the program goes through a conditional to check if the number of cards is greater than 0 (if there is at least one card). If it passes the condition, it deletes the question and the answer of currently displayed card from the arrays. Then, it decreases the total length of a sequence by 1.
        If the current card was the last card, it decreases the current number by 1, which will be the number of the last card. If the total length is 0, which means there is no card left, it chooses a random emoji and display that there is no card. If not, it displays the current card with a question and updates the current page number and the total page number at the bottom of the page.

[ToDo List]

Adding ToDo Items

        This flowchart shows how new items are added to the ToDo list.

When the 'plus' button is clicked, the program increases the variable of number of items by 1. Then, it calls the function to save the input values before updating HTML.

        In the following function, it initialize an empty array to store the values. After declaring variables to get elements of title, description, and checkbox from HTML, it starts a loop for the items in the ToDo List. For every items, the function push an array of values from input fields of title and description and the status of checkbox as a boolean into the array created earlier.

        Then, the program append the formatted HTML for an item to the container. Using a fade-in transition, it displays the new item in the list. Lastly, the program calls the function to overwrite the saved values on the initialized input fields of every item. This function also loops all the items in the list to write the value with the data collected in the array during the previous function. After it writes all the values of items, the program ends adding a new item.
